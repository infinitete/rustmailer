# Git Hooks Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 为仓库增加原生 Git hooks，在提交前执行前后端质量检查，并校验提交信息符合 Conventional Commits。

**Architecture:** 通过 `git config core.hooksPath .githooks` 把仓库内脚本注册为 Git hooks。`pre-commit` 负责执行前后端校验链路，`commit-msg` 负责验证提交信息格式，安装入口统一由 `scripts/setup-git-hooks.sh` 提供。

**Tech Stack:** Git hooks, POSIX shell, Cargo, npm

---

### Task 1: Create the hook installation entrypoint

**Files:**
- Create: `scripts/setup-git-hooks.sh`

**Step 1: Define the installer behavior**

Make the script resolve the repository root and run `git config core.hooksPath .githooks` from there.

**Step 2: Print a clear success message**

Include the configured hooks path and a short reminder that hooks are now active for this local clone.

**Step 3: Verify the installer**

Run: `./scripts/setup-git-hooks.sh`
Expected: exit code `0` and a message confirming `.githooks` is registered.

**Step 4: Verify Git config**

Run: `git config --get core.hooksPath`
Expected: output is exactly `.githooks`.

### Task 2: Implement the pre-commit quality gate

**Files:**
- Create: `.githooks/pre-commit`

**Step 1: Write the shell script**

Add a script that enters the repo root and runs:

- `cd server && cargo fmt --check`
- `cd server && cargo test`
- `cd frontend && npm run lint`
- `cd frontend && npm run build`

**Step 2: Add dependency guards**

Fail early with a readable message if `frontend/node_modules` is missing.

**Step 3: Verify the hook manually**

Run: `.githooks/pre-commit`
Expected: the script executes the checks in order and exits non-zero on the first failing command.

### Task 3: Implement commit message validation

**Files:**
- Create: `.githooks/commit-msg`

**Step 1: Validate the first line only**

Read the commit message file and extract the first line.

**Step 2: Enforce Conventional Commits**

Accept types `feat|fix|docs|style|refactor|test|chore|build|ci|perf|revert`, optional scope, optional `!`, and a required subject.

**Step 3: Allow Git-generated special cases**

Permit `Merge ...`, `Revert ...`, `fixup! ...`, and `squash! ...`.

**Step 4: Verify invalid message fails**

Run: `tmpfile=$(mktemp) && printf 'bad message\n' > "$tmpfile" && .githooks/commit-msg "$tmpfile"`
Expected: exit code non-zero with a format hint.

**Step 5: Verify valid message passes**

Run: `tmpfile=$(mktemp) && printf 'feat(frontend): add pre-commit quality gate\n' > "$tmpfile" && .githooks/commit-msg "$tmpfile"`
Expected: exit code `0` and no error output.

### Task 4: Document usage

**Files:**

**Step 1: Add setup instructions**

Document the one-time install step and list what each hook checks.

**Step 2: Add troubleshooting notes**

Explain that frontend dependencies must be installed before the hook can pass.

**Step 3: Verify docs match reality**

Re-read the scripts and ensure commands and paths in the README are exact.

**Files:**
